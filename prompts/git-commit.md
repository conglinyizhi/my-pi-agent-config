---
description: 根据 Git 差异生成符合约定式提交的中文 commit message
argument-hint: "[补充说明]"
---

请根据当前仓库的 Git 差异，生成符合《约定式提交》规范的中文 commit message。

执行步骤：

1. 先获取暂存区差异；如果输出为空，则获取工作区差异。
2. 分析改动的类型、范围和目的。如果需要更多上下文你可以自行读取对应的文件
3. 生成一行提交信息，格式：`<type>(<scope>): <中文描述>`
   - 描述控制在 30 字以内
   - scope优先使用修改模块的短英文名称
   - 无法确定scope的，可以省略该部分
   - 涉及破坏性修改的，需要添加英文感叹号
4. 对于复杂变动需要补充说明，将其作为 commit body 追加在标题之后。
5. 如果规划的 body 内容和描述内容重合度过高，就不要放 body 部分

用户补充信息：<user-say>$@</user-say>

示例输出：

```text
feat(cli.go): 命令行 -t 参数用来启动测试能力。

完善测试能力，当启用 -t 时，程序将会启动新的特性分支，发起测试请求，无论成功与否，返回测试结果后立刻退出

close #3
```

或者轻量版本：`feat(cli.go): 命令行 -t 参数用来启动测试能力。`

完成生成后发起 git commit 提交即可，注意提交范围，以及确保输出的 commit message 部分为中文。任务正式完成后，简单回复用户即可
