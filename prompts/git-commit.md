请根据当前仓库的 Git 差异，生成符合《约定式提交》规范的中文 commit message。

<user-say alt="用户补充信息">$@</user-say>

模板：

```text
<类型>[可选 范围]: <描述>

[可选 正文]

[可选 脚注]
```

执行要点：

- 同时获取差异和工作区差异，优先发起暂存区的内容提交；所有差异在工作区时记得看看没有追踪但可能存在的新文件
- 根据上下文和 user-say 分析改动的类型、范围和目的；可自行读取额外文件
- 描述控制在 30 字以内
- 范围优先使用修改模块的短英文名称，无法确定范围的，可以省略该部分
- 涉及破坏性修改的，需要添加英文感叹号
- 对于复杂变动需要补充说明，将其作为 commit 正文 追加在标题之后。
- 如果规划的正文内容和描述内容重合度过高，就不要放 正文 部分；当用户补充了无法从源码变动中看出的内容时，应将 user-say 内容写入正文内

示例输出：

```text
fix: 防止请求竞态

引入请求 ID 与最新请求的引用。忽略非来自最新请求的响应。

移除原本用于缓解竞态问题的超时机制，现已不再需要。

Refs: #123
```

或者轻量版本：`feat(cli.go): 命令行 -t 参数用来启动测试能力。`

完成生成后发起 git commit 提交即可，注意提交范围，以及确保输出的 commit message 部分为中文。任务正式完成后，简单回复用户即可
如果遇到 git hook 阻止，停止并简述原因后视为任务完成
当 user-say 没有额外说明，仅允许进行一次提交
若暂存区提交后工作区（或者执行完 user-say 提供的命令）仍然有修改，用文字的方式提示用户（禁止在此步骤调用工具）同时视为任务完成
